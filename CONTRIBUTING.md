# 共建指南

欢迎一起完善 OpenClaw 搭建手记项目！

## 如何参与

我们欢迎各种形式的贡献，无论是修正文档错误、补充遗漏内容，还是分享你的搭建经验。

---

## 报告问题

如果你在搭建OpenClaw时遇到问题：

### 使用GitHub Issues

1. 访问 [Issues页面](https://github.com/dctongsheng/build_openclaw_from_0_1_living/issues)
2. 点击 "New Issue"
3. 选择合适的模板（经验分享、Bug报告、内容建议、问题咨询）
4. 填写详细信息并提交

### 提供有用的信息

为了更好地帮助你，请提供：

- **操作系统版本**（如 macOS 14.5, Ubuntu 22.04）
- **Node.js版本**（运行 `node -v`）
- **OpenClaw版本**（如果已安装）
- **详细的错误信息**（完整的错误堆栈）
- **复现步骤**（如何触发这个问题）
- **截图**（如果问题涉及UI）
- **相关配置**（如 `openclaw.json`，注意隐藏敏感信息）

## 提交改进

如果你想改进文档内容或修正错误：

### 1. Fork本仓库

点击 GitHub 页面右上角的 "Fork" 按钮

### 2. 创建你的特性分支

```bash
git checkout -b feature/AmazingFeature
```

### 3. 提交你的修改

```bash
git commit -m 'Add some AmazingFeature'
```

### 4. 推送到分支

```bash
git push origin feature/AmazingFeature
```

### 5. 开启一个Pull Request

访问你Fork的仓库页面，点击 "Compare & pull request"

## Pull Request 指南

### PR标题

使用清晰、描述性的标题：

- ✅ 好的：`Fix: 修正WhatsApp配置中的API密钥路径`
- ✅ 好的：`Add: 添加Docker部署方式说明`
- ❌ 不好：`update doc`
- ❌ 不好：`fix bug`

### PR描述

在PR描述中说明：

- **做了什么修改**（简要描述）
- **为什么做这个修改**（原因说明）
- **如何验证**（如果适用）
- **相关Issue**（如 `Closes #123`）

### 代码风格

- 使用Markdown语法
- 保持一致的格式
- 添加适当的空行分隔章节
- 使用代码块展示命令和配置
