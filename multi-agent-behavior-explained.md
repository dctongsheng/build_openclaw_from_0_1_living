## 🔍 正确理解：Agent vs Session

### Agent（员工）= 永久存在

```
┌──────────────────────────────────────┐
│         Agent 员工                    │
├──────────────────────────────────────┤
│ 📚 研究助手 - 一直存在，随时可调用     │
│ 🌟 生活助手 - 一直存在，随时可调用     │
│ 🦐 小虾米    - 一直存在，主协调者      │
└──────────────────────────────────────┘
        ↓ 随时可以调用
```

**特点：**
- Agent 永久存在，不会消失
- 可以反复调用同一个 Agent
- Agent 之间可以互相通信
- Agent 有独立的工作区和配置

### Session（会话）= 临时任务

```
小虾米 → sessions_spawn({agentId: "research"})
        ↓
   创建子会话
        ↓
┌──────────────────────────────────────┐
│     Session 子会话                    │
├──────────────────────────────────────┤
│ 状态：running → 任务执行中             │
│ 状态：done    → 任务完成               │
│ 注意：done 状态下仍可继续对话！        │
└──────────────────────────────────────┘
```

**状态说明：**

| 状态 | 含义 | 可以对话？ |
|------|------|-----------|
| `running` | 任务正在执行 | ✅ 可以 |
| `done` | 任务已完成 | ✅ 可以（不是"下班"）|

**重要：**
- `done` 状态 = 任务完成，不是会话结束
- `done` 状态的会话仍可以继续对话
- 可以继续向该会话提问
- 会话保持活跃，直到超时或手动关闭

## 💬 小虾米的回复为什么这样说？

### "临时工模式"的理解

小虾米说：
```
这俩员工是按需调用的临时工模式，有活才叫，没活就解散
```

**这个理解部分正确：**

✅ **正确的部分：**
- 员工是按需调用的
- 有任务才调用 sessions_spawn
- 调用后创建临时会话

❌ **需要纠正的部分：**
- 员工不是"临时工"
- Agent（员工）永久存在
- 只是会话（Session）是临时的
- `done` 状态不是"解散"，是"任务完成"

### "又变回光杆司令了"的理解

小虾米说：
```
当前状态：0 个在岗员工，又变回光杆司令了
```

**这个理解需要澄清：**

❌ **错误理解：**
- 会话状态是 done = 员工下班了
- 没有运行中的会话 = 没有员工

✅ **正确理解：**
- 会话状态是 done = 任务完成（仍可继续对话）
- 没有运行中的会话 = 当前没有活跃的任务
- 员工一直都在，随时可以再次调用

**类比：**
```
员工 = 你的医生
会话 = 你的诊疗记录

医生（员工）一直都在医院
诊疗记录（会话）创建后状态变为"已完成"
但你可以继续找同一个医生看病
也可以创建新的诊疗记录

"没有正在进行的诊疗" ≠ "医生下班了"
```
