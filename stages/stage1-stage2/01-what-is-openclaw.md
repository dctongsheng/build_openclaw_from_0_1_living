## OpenClaw 是什么

**OpenClaw** 是一个自托管的 AI Agent Gateway，它可以：

- 将各种聊天应用连接到强大的 AI 编码 Agent
- 作为一个统一的网关，管理所有 AI 交互
- 支持多平台、多 Agent、多会话的复杂场景

### 定义

OpenClaw 是一个**代理原生的**、**自托管的** Gateway，用于将聊天应用连接到 AI Agent。

### 核心价值

1. **数据隐私** - 所有数据都在你的控制之下
2. **多通道支持** - 一个网关支持所有主流聊天平台
3. **代理原生** - 专为 AI Agent 设计，不是简单的聊天机器人
4. **完全开源** - 自由定制和扩展

## 核心特性

### 1. 多通道支持

OpenClaw 支持超过 15 种消息通道：

| 通道类型 | 支持状态 | 配置难度 |
|---------|---------|---------|
| WhatsApp | ✅ 完全支持 | 中等 |
| Telegram | ✅ 完全支持 | 简单 |
| Discord | ✅ 完全支持 | 简单 |
| Slack | ✅ 完全支持 | 中等 |
| Signal | ✅ 完全支持 | 简单 |
| iMessage | ✅ 完全支持 | 复杂 |
| Google Chat | ✅ 完全支持 | 中等 |
| Microsoft Teams | ✅ 完全支持 | 中等 |
| IRC | ✅ 完全支持 | 简单 |
| LINE | ✅ 完全支持 | 中等 |
| Matrix | ✅ 完全支持 | 中等 |
| Mattermost | ✅ 完全支持 | 中等 |

### 2. 代理原生

OpenClaw 的 Agent 运行时基于 **pi-mono**，具有以下特性：

- **Agent Loop** - 完整的 Agent 生命周期管理
- **工具系统** - 丰富的内置工具，支持自定义扩展
- **会话管理** - 多会话、多 Agent 路由
- **流式响应** - 实时流式传输 AI 响应

### 3. 灵活的模型支持

支持多个 LLM Provider：

- OpenAI (GPT-4, GPT-3.5)
- Anthropic (Claude 系列)
- AWS Bedrock
- OpenRouter
- Google Gemini
- 其他兼容 OpenAI API 的服务

### 4. 安全沙箱

- **Sandbox 模式** - 限制 Agent 的操作权限
- **Elevated 模式** - 特殊操作需要显式授权
- **工具策略** - 细粒度控制可用工具
- **白名单机制** - 控制谁可以与 Agent 交互

### Session 和 Routing

1. **Session** - 一次完整的对话会话
   - 存储在 `~/.openclaw/sessions/` 目录
   - JSONL 格式，便于追溯和分析
   - 支持 Session Pruning（修剪旧内容）

2. **Routing** - 消息路由机制
   - 根据 Agent 配置路由到不同的 Agent
   - 支持多 Agent 协作
   - 支持动态 Agent 选择

## 使用场景

### 场景 1：个人 AI 助手

**描述**：在多个聊天平台上拥有一个统一的 AI 助手

**特点**：
- 一对一对话
- 代码辅助
- 信息检索
- 日程管理

**配置**：
```json
{
  "channels": {
    "whatsapp": {
      "allowFrom": ["+8613800138000"]
    },
    "telegram": {
      "allowFrom": ["@my_telegram_username"]
    }
  }
}
```

### 场景 2：团队协作机器人

**描述**：团队共享的 AI 助手，帮助团队提高生产力

**特点**：
- 群组集成
- 知识共享
- 任务协作
- 代码审查

**配置**：
```json
{
  "channels": {
    "discord": {
      "guilds": {
        "allow": ["123456789012345678"]
      }
    }
  }
}
```

### 场景 3：客户服务自动化

**描述**：AI 客户服务机器人，处理常见问题和简单任务

**特点**：
- 多渠道接入
- 知识库检索
- 工单创建
- 人工转接

**配置**：
```json
{
  "agents": {
    "list": [
      {
        "id": "support",
        "model": "openai/gpt-4o",
        "tools": {
          "profile": "messaging"
        }
      }
    ]
  }
}
```

### 场景 4：开发工作流增强

**描述**：集成到开发工作流中，自动化重复性任务

**特点**：
- CI/CD 集成
- 自动测试
- 文档生成
- Bug 分析

**实现**：通过 Cron 工具实现定时任务

## 与 SaaS AI 服务的区别

| 特性 | OpenClaw | SaaS AI 服务 |
|-----|---------|-------------|
| **数据隐私** | 完全自托管，数据本地化 | 数据上传到第三方服务器 |
| **定制能力** | 完全可定制 | 受限于平台功能 |
| **成本** | 仅 API 调用成本 | 平台费用 + API 成本 |
| **部署方式** | 本地/私有云 | 云服务 |
| **多平台支持** | 统一网关，全平台 | 通常单平台 |
| **扩展性** | 开源，可扩展 | 受限于平台 |

### 何时选择 OpenClaw？

✅ **选择 OpenClaw** 如果你：
- 注重数据隐私和安全
- 需要支持多个聊天平台
- 希望深度定制 AI Agent 行为
- 有技术能力进行自托管
- 需要集成到现有系统

❌ **考虑 SaaS 服务** 如果你：
- 没有技术维护能力
- 只需要简单的聊天机器人
- 不关心数据隐私
- 希望快速上手
